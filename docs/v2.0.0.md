# Sky Flux CMS — V2.0.0 版本规划

**目标**：在 V1.0 稳定版基础上引入高级功能，增强内容创作体验和系统扩展性。

**状态**：规划中（V1.0 完成后启动）

**来源**：PRD 3.2 节 "V1.1 未来功能"

---

## 前置条件

- V1.0.0 全部功能完成并稳定运行
- 核心 API 接口冻结（REST API 不做 Breaking Change）
- E2E 测试覆盖率 > 80%
- 生产环境部署验证通过

---

## 功能清单

### 2.1 AI 辅助内容

**概述**：利用 AI 能力提升内容创作效率和质量。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| AI 文章摘要 | 基于文章正文自动生成摘要（excerpt），支持一键填入 | P1 |
| SEO 优化建议 | 分析文章内容，给出标题、描述、关键词优化建议 | P1 |
| 内容质量评分 | 可读性、SEO 友好度、内容长度等维度评分 | P2 |
| 智能标签推荐 | 根据文章内容推荐相关标签 | P2 |

**技术方向**：
- 后端 Service 层集成 LLM API（OpenAI / Anthropic / 本地模型）
- 异步处理（避免阻塞编辑体验）
- 结果缓存（相同内容不重复调用）

### 2.2 Webhook 通知

**概述**：支持将 CMS 事件推送至第三方系统。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Webhook 注册 | 管理 Webhook URL、事件类型、密钥 | P1 |
| 内容事件触发 | 文章发布/更新/删除时触发 Webhook | P1 |
| 重试机制 | 失败后指数退避重试（最多 3 次） | P1 |
| 事件日志 | 记录每次 Webhook 投递的状态和响应 | P1 |
| 签名验证 | HMAC-SHA256 请求签名，接收方可验证来源 | P1 |

**技术方向**：
- 异步队列投递（Redis/PostgreSQL LISTEN-NOTIFY）
- `sfc_site_webhooks` + `sfc_site_webhook_logs` 表
- 管理 UI：Webhook 配置 + 投递日志查看

### 2.3 GraphQL API

**概述**：在现有 REST API 基础上提供 GraphQL 查询接口，满足复杂前端数据需求。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Schema 自动生成 | 基于 Model 层生成 GraphQL Schema | P1 |
| 嵌套查询 | 文章 + 分类 + 标签 + 评论 + 作者一次性查询 | P1 |
| 分页 & 筛选 | Relay-style cursor 分页 + 灵活筛选 | P1 |
| 权限集成 | 复用 REST API 的 RBAC + API Key 鉴权 | P1 |
| Schema 文档 | 自动生成 GraphQL Playground / Docs | P2 |

**技术方向**：
- Go GraphQL 库选型（gqlgen / graphql-go）
- 复用现有 Service 层和 Repository 层
- 独立路由挂载（`/graphql`），与 REST 共存

### 2.4 插件系统

**概述**：提供插件注册与生命周期管理机制，允许第三方扩展 CMS 功能。

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 插件注册 | 插件 manifest 声明（名称、版本、依赖、Hook 列表） | P1 |
| Hook 机制 | 内容发布前/后、评论提交前/后等扩展点 | P1 |
| 插件 API | 插件可注册自定义 API 端点 | P2 |
| 插件管理 UI | 安装/卸载/启用/禁用插件 | P2 |
| 插件市场 | 在线浏览和安装社区插件（V2.1） | P3 |

**技术方向**：
- Go plugin 包 或 基于 RPC 的进程隔离模型
- Hook 注册表（事件 → Handler 列表）
- 插件沙箱（限制文件系统和网络访问）

## 与 V1.0 的关系

- V2.0 **不破坏** V1.0 的 REST API 接口契约
- V2.0 的 GraphQL API 是 REST API 的**补充**，而非替代
- 插件系统通过 Hook 机制扩展，**不修改**核心代码路径
- AI 功能为**可选增强**，不影响基础内容管理流程
